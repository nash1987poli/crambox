Purpose
This document defines the mathematical architecture of Crambox.
The objective is to ensure:
· Correct answers · Verified solutions · Unlimited question generation · Offline operation · ZIMSEC alignment · Exam readiness accuracy Mathematical correctness is a non-negotiable requirement.
Core Philosophy
Never trust generated answers.
Always verify generated answers.
The architecture must separate:
1. Question Generation 2. Mathematical Solving 3. Verification 4. Readiness Calculation These are independent systems.
